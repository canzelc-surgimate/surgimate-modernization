Surgimate Modernization

The modern experience ships in light mode by default — the look most customers will use day to day. Dark mode is optional (Profile → Appearance). Classic UI stays available behind a per-client feature flag until each site is ready to switch.

Toggle below to compare Classic, Modern Light, and Modern Dark — all screenshots captured from the running app.

Navigation

One navbar instead of two

Classic Surgimate uses a white utility bar (Welcome, Tutorials, Support, Release Notes) stacked above a dark tab bar (Home, Overview, Reports, Dashboard, Agenda…). Modern UI consolidates everything into a single 68px navbar with Lucide icons, rounded patient search, and the Ask Surgi entry point.

  • Overview tab removed — surgeries search lives on Home as the Overview widget (modern light is the default layout customers see).
  • Reports tab removed — reports move to Dashboards → Report Library.
  • Tutorials, Support, Release Notes — moved into the Profile dropdown (Help section), not lost.
  • Admin — no longer a top-level tab; Admin Dashboard is in Profile → Client Admin.

Profile menu

Profile dropdown — account, admin & help in one place

Classic scatters links across the top utility bar and a minimal Welcome menu. Modern groups everything into a structured dropdown with labeled sections.

  • Your Account — Account Settings, Notification Rules, Configure 2FA.
  • Client Admin — Admin Dashboard (moved from the old ADMIN tab), Backend Configurations, Impersonate, Dashboards & Reporting for SurgiAdmin users.
  • Help — Tutorials, Support, Release Notes (previously always visible in the classic header).
  • Appearance — Light / Dark toggle (modern only; light is the default).
  • Log out — bottom of the menu.

Home · Launch Pad

From static charts to a customizable Launch Pad

Classic Home shows fixed chart widgets (Upcoming Events, Reports, Cancellation Details) in a rigid two-column layout — same for every user. Modern Home is a personal Launch Pad: drag, resize, add/remove widgets. Each widget previews live data and drills into its full tab (Agenda, Dashboards, or inline Overview search).

  • Overview widget — replaces the standalone Overview tab (saved views, filters, CSV).
  • Agenda widget — For You / Watchlist task preview → full Agenda tab.
  • Dashboard widgets — curated KPI charts → Analytics Hub.
  • Gear menu — “Add or remove widgets” and “Move & resize widgets” (see below).

Dashboards

Native Analytics Hub vs legacy dashboards

Classic clients with analytics enabled may still see embedded Apache Superset iframes — a different visual language from the rest of Surgimate. When the modern flag is on, Dashboards becomes the native Analytics Hub: same teal brand, typography, rounded cards, and light/dark tokens as Home and Agenda. No context switch to an external UI.

  • Report Library replaces the old Reports tab.
  • Saved views per user, per client, per dashboard.
  • Case list concepts fold into Overview saved views on Home.
  • Hub landing cards drill into Executive Scorecard, Volume Tracker, Scheduling Health, and more.

Surgi AI

Ask Surgi — brand new in modern UI

There is no classic equivalent. Ask Surgi opens a slide-in panel from any modern screen. Phase 1 is conversational and read-only (AWS Bedrock) — ask questions about your data without leaving context. Available in both light and dark themes.

Rollout

Feature flag — opt in per client

Modernization is controlled by modern_ui_ux_enabled in Admin → Clients → Client Settings. When OFF, users see today's classic develop UI unchanged. When ON, the modern shell activates for that client — starting in light mode unless a user chooses dark in Profile → Appearance.

  • Checkbox: “Modern UI/UX: Enable Surgimate modernizations”
  • Phased beta: enable for pilot clients, gather feedback, then expand.
  • Classic and modern share the same data, permissions, and workflows — only the surfaces change.
